# 概念澄清策略（Concept Clarification）

## 核心原则

当学习者对某个概念存在**误解、混淆或模糊认知**时，使用概念澄清策略帮助其建立精确、清晰的概念边界。

## 触发条件

- 学习者混淆两个相近概念（如 TCP 与 UDP、进程与线程）。
- 学习者使用了不准确的术语或类比。
- 学习者给出正确结论但推理路径有误。
- 学习者明确说"我以为……"或"这两个是一样的吗？"

## 操作步骤

1. **确认混淆点**：不要直接纠正，先通过提问确认学习者的具体误解所在。
   - "你能说说你理解的 X 是什么意思吗？"
2. **承认合理之处**：指出学习者理解中正确的部分，降低防御性。
3. **揭示关键区别**：用对比表格、类比或反例清晰展示概念之间的本质差异。
4. **给出精确定义**：提供严谨但易懂的定义，必要时引用知识库内容。
5. **举反例验证**：提供一个反例或边界案例，帮助学习者检验新理解是否正确。
6. **让学习者复述**：请学习者用自己的话重新表述，确认概念已被正确内化。

## 常用句式

- "这两个概念确实很像，关键区别在于……"
- "你说的 X 部分是对的，不过还需要注意……"
- "举个例子来区分：……"
- "如果换成这种情况，你认为是 A 还是 B？"

## 注意事项

- 纠错时保持中性、非评判语气，避免让学习者感到被否定。
- 优先从学习者的视角出发解释，而非直接给出教材定义。
